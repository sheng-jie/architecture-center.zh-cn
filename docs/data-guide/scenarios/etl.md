# <a name="extract-transform-and-load-etl"></a><span data-ttu-id="2b486-102">提取、转换和加载 (ETL)</span><span class="sxs-lookup"><span data-stu-id="2b486-102">Extract, transform, and load (ETL)</span></span>

<span data-ttu-id="2b486-103">组织面临的常见难题是如何从多个源收集多种格式的数据，并将其转移到一个或多个数据存储。</span><span class="sxs-lookup"><span data-stu-id="2b486-103">A common problem that organizations face is how to gathering data from multiple sources, in multiple formats, and move it to one or more data stores.</span></span> <span data-ttu-id="2b486-104">目标的数据存储类型可能与源不同，并且格式通常不同，或者在将数据载入其最终目标之前，需要对其进行整理或清理。</span><span class="sxs-lookup"><span data-stu-id="2b486-104">The destination may not be the same type of data store as the source, and often the format is different, or the data needs to be shaped or cleaned before loading it into its final destination.</span></span>

<span data-ttu-id="2b486-105">过去几年以来，行业中已开发了多种工具、服务和过程来解决这些难题。</span><span class="sxs-lookup"><span data-stu-id="2b486-105">Various tools, services, and processes have been developed over the years to help address these challenges.</span></span> <span data-ttu-id="2b486-106">不管采用哪种过程，往往都需要协调工作，并在数据管道中应用某种级别的数据转换。</span><span class="sxs-lookup"><span data-stu-id="2b486-106">No matter the process used, there is a common need to coordinate the work and apply some level of data transformation within the data pipeline.</span></span> <span data-ttu-id="2b486-107">以下部分重点介绍执行这些任务的常用方法。</span><span class="sxs-lookup"><span data-stu-id="2b486-107">The following sections highlight the common methods used to perform these tasks.</span></span>

## <a name="extract-transform-and-load-etl"></a><span data-ttu-id="2b486-108">提取、转换和加载 (ETL)</span><span class="sxs-lookup"><span data-stu-id="2b486-108">Extract, transform, and load (ETL)</span></span>

<span data-ttu-id="2b486-109">提取、转换和加载 (ETL) 是一个数据管道，用于从各种源收集数据、根据业务规则转换数据，然后将其载入目标数据存储。</span><span class="sxs-lookup"><span data-stu-id="2b486-109">Extract, transform, and load (ETL) is a data pipeline used to collect data from various sources, transform the data according to business rules, and load it into a destination data store.</span></span> <span data-ttu-id="2b486-110">ETL 中的转换工作在专用引擎中发生，通常涉及到使用临时表来暂时保存正在转换、并最终要载入其目标的数据。</span><span class="sxs-lookup"><span data-stu-id="2b486-110">The transformation work in ETL takes place in a specialized engine, and often involves using staging tables to temporarily hold data as it is being transformed and ultimately loaded to its destination.</span></span>

<span data-ttu-id="2b486-111">数据转换通常涉及到各种操作，例如筛选、排序、聚合、联接数据、清理数据、删除重复数据和验证数据。</span><span class="sxs-lookup"><span data-stu-id="2b486-111">The data transformation that takes place usually involves various operations, such as filtering, sorting, aggregating, joining data, cleaning data, deduplicating, and validating data.</span></span>

![提取-转换-加载 (ETL) 过程](./images/etl.png)

<span data-ttu-id="2b486-113">通常，三个 ETL 阶段是同时运行的，以节省时间。</span><span class="sxs-lookup"><span data-stu-id="2b486-113">Often, the three ETL phases are run in parallel to save time.</span></span> <span data-ttu-id="2b486-114">例如，在提取数据时，转换过程可以处理已收到的数据并做好加载数据的准备，加载过程可以开始处理已准备好的数据，而无需等待整个提取过程完成。</span><span class="sxs-lookup"><span data-stu-id="2b486-114">For example, while data is being extracted, a transformation process could be working on data already received and prepare it for loading, and a loading process can begin working on the prepared data, rather than waiting for the entire extraction process to complete.</span></span>

## <a name="extract-load-and-transform-elt"></a><span data-ttu-id="2b486-119">提取、加载和转换 (ELT)</span><span class="sxs-lookup"><span data-stu-id="2b486-119">Extract, load, and transform (ELT)</span></span>

<span data-ttu-id="2b486-120">提取、加载和转换 (ELT) 与 ETL 的唯一不同之处在于转换的发生位置。</span><span class="sxs-lookup"><span data-stu-id="2b486-120">Extract, load, and transform (ELT) differs from ETL solely in where the transformation takes place.</span></span> <span data-ttu-id="2b486-121">在 ELT 管道中，转换发生在目标数据存储中。</span><span class="sxs-lookup"><span data-stu-id="2b486-121">In the ELT pipeline, the transformation occurs in the target data store.</span></span> <span data-ttu-id="2b486-122">在不使用单独转换引擎的情况下，将使用目标数据存储的处理功能来转换数据。</span><span class="sxs-lookup"><span data-stu-id="2b486-122">Instead of using a separate transformation engine, the processing capabilities of the target data store are used to transform data.</span></span> <span data-ttu-id="2b486-123">这就从管道中删除了转换引擎，从而简化了体系结构。</span><span class="sxs-lookup"><span data-stu-id="2b486-123">This simplifies the architecture by removing the transformation engine from the pipeline.</span></span> <span data-ttu-id="2b486-124">此方法的另一个好处是，缩放目标数据存储也会缩放 ELT 管道性能。</span><span class="sxs-lookup"><span data-stu-id="2b486-124">Another benefit to this approach is that scaling the target data store also scales the ELT pipeline performance.</span></span> <span data-ttu-id="2b486-125">但是，仅当目标系统足够强大，可以有效转换数据时，ELT 才能正常工作。</span><span class="sxs-lookup"><span data-stu-id="2b486-125">However, ELT only works well when the target system is powerful enough to transform the data efficiently.</span></span>

![提取-加载-转换 (ELT) 过程](./images/elt.png)

<span data-ttu-id="2b486-127">ELT 的典型用例属于大数据领域。</span><span class="sxs-lookup"><span data-stu-id="2b486-127">Typical use cases for ELT fall within the big data realm.</span></span> <span data-ttu-id="2b486-128">例如，首先可将所有源数据提取到 Hadoop 分布式文件系统 (HDFS) 或 Azure Data Lake Store 等可缩放存储中的平面文件。</span><span class="sxs-lookup"><span data-stu-id="2b486-128">For example, you might start by extracting all of the source data to flat files in scalable storage such as Hadoop distributed file system (HDFS) or Azure Data Lake Store.</span></span> <span data-ttu-id="2b486-129">然后，可以使用 Spark、Hive 或 PolyBase 等技术查询源数据。</span><span class="sxs-lookup"><span data-stu-id="2b486-129">Technologies such as Spark, Hive, or PolyBase can then be used to query the source data.</span></span> <span data-ttu-id="2b486-130">ELT 的关键之处在于，用于执行转换的数据存储是最终要在其中使用数据的同一数据存储。</span><span class="sxs-lookup"><span data-stu-id="2b486-130">The key point with ELT is that the data store used to perform the transformation is the same data store where the data is ultimately consumed.</span></span> <span data-ttu-id="2b486-131">此数据存储直接从可缩放的存储读取，而不是将数据载入其自身的专有存储。</span><span class="sxs-lookup"><span data-stu-id="2b486-131">This data store reads directly from the scalable storage, instead of loading the data into its own proprietary storage.</span></span> <span data-ttu-id="2b486-132">此方法跳过了 ETL 中存在的数据复制步骤，针对大型数据集运行时可能很耗时。</span><span class="sxs-lookup"><span data-stu-id="2b486-132">This approach skips the data copy step present in ETL, which can be a time consuming operation for large data sets.</span></span>

<span data-ttu-id="2b486-133">在实践中，目标数据存储是使用 Hadoop 群集（使用 Hive 或 Spark）的[数据仓库](./data-warehousing.md)，或 SQL 数据仓库。</span><span class="sxs-lookup"><span data-stu-id="2b486-133">In practice, the target data store is a [data warehouse](./data-warehousing.md) using either a Hadoop cluster (using Hive or Spark) or a SQL Data Warehouse.</span></span> <span data-ttu-id="2b486-134">一般情况下，架构在查询时叠加在平面文件数据上，并存储为表，因此，可以像查询数据存储中的其他任何数据一样查询这些数据。</span><span class="sxs-lookup"><span data-stu-id="2b486-134">In general, a schema is overlaid on the flat file data at query time and stored as a table, enabling the data to be queried like any other table in the data store.</span></span> <span data-ttu-id="2b486-135">这些表称为外部表，因为数据不在数据存储本身管理的存储中，而是在某个外部可缩放存储中。</span><span class="sxs-lookup"><span data-stu-id="2b486-135">These are referred to as external tables because the data does not reside in storage managed by the data store itself, but on some external scalable storage.</span></span> 

<span data-ttu-id="2b486-136">数据存储只管理数据的架构，并在读取时应用架构。</span><span class="sxs-lookup"><span data-stu-id="2b486-136">The data store only manages the schema of the data and applies the schema on read.</span></span> <span data-ttu-id="2b486-137">例如，使用 Hive 的 Hadoop 群集会描述一个 Hive 表，其中的数据源实际上是 HDFS 中一组文件的路径。</span><span class="sxs-lookup"><span data-stu-id="2b486-137">For example, a Hadoop cluster using Hive would describe a Hive table where the data source is effectively a path to a set of files in HDFS.</span></span> <span data-ttu-id="2b486-138">在 SQL 数据仓库中，PolyBase 可以实现相同的结果 &mdash; 针对数据库本身外部存储的数据创建一个表。</span><span class="sxs-lookup"><span data-stu-id="2b486-138">In SQL Data Warehouse, PolyBase can achieve the same result &mdash; creating a table against data stored externally to the database itself.</span></span> <span data-ttu-id="2b486-139">加载源数据后，可以使用数据存储的功能处理外部表中的数据。</span><span class="sxs-lookup"><span data-stu-id="2b486-139">Once the source data is loaded, the data present in the external tables can be processed using the capabilities of the data store.</span></span> <span data-ttu-id="2b486-140">在大数据方案中，这意味着数据存储必须能够执行大规模并行处理 (MPP)，将数据分解成较小区块，并跨多台计算机并行分配区块处理负载。</span><span class="sxs-lookup"><span data-stu-id="2b486-140">In big data scenarios, this means the data store must be capable of massively parallel processing (MPP), which breaks the data into smaller chunks and distributes processing of the chunks across multiple machines in parallel.</span></span>

<span data-ttu-id="2b486-141">ELT 管道的最后一个阶段通常是将源数据转换为最终格式，对于需要支持的查询类型，此格式更为有效。</span><span class="sxs-lookup"><span data-stu-id="2b486-141">The final phase of the ELT pipeline is typically to transform the source data into a final format that is more efficient for the types of queries that need to be supported.</span></span> <span data-ttu-id="2b486-142">例如，可将数据分区。</span><span class="sxs-lookup"><span data-stu-id="2b486-142">For example, the data may be partitioned.</span></span> <span data-ttu-id="2b486-143">此外，ELT 可以使用 Parquet 等优化的存储格式，以纵栏表形式存储面向行的数据，并提供优化的索引。</span><span class="sxs-lookup"><span data-stu-id="2b486-143">Also, ELT might use optimized storage formats like Parquet, which stores row-oriented data in a columnar fashion and providess optimized indexing.</span></span>

## <a name="data-flow-and-control-flow"></a><span data-ttu-id="2b486-151">数据流和控制流</span><span class="sxs-lookup"><span data-stu-id="2b486-151">Data flow and control flow</span></span>

<span data-ttu-id="2b486-152">在数据管道的上下文中，控制流确保有序处理一组任务。</span><span class="sxs-lookup"><span data-stu-id="2b486-152">In the context of data pipelines, the control flow ensures orderly processing of a set of tasks.</span></span> <span data-ttu-id="2b486-153">若要实施这些任务的正确处理顺序，可以使用优先约束。</span><span class="sxs-lookup"><span data-stu-id="2b486-153">To enforce the correct processing order of these tasks, precedence constraints are used.</span></span> <span data-ttu-id="2b486-154">可将这些约束视为工作流关系图中的连接器，如下图所示。</span><span class="sxs-lookup"><span data-stu-id="2b486-154">You can think of these constraints as connectors in a workflow diagram, as shown in the image below.</span></span> <span data-ttu-id="2b486-155">每个任务都有结果，例如成功、失败或完成。</span><span class="sxs-lookup"><span data-stu-id="2b486-155">Each task has an outcome, such as success, failure, or completion.</span></span> <span data-ttu-id="2b486-156">只有在前面的任务已完成并生成了结果时，后面的任务才会启动处理。</span><span class="sxs-lookup"><span data-stu-id="2b486-156">Any subsequent task does not initiate processing until its predecessor has completed with one of these outcomes.</span></span>

<span data-ttu-id="2b486-157">控制流以任务的形式执行数据流。</span><span class="sxs-lookup"><span data-stu-id="2b486-157">Control flows execute data flows as a task.</span></span> <span data-ttu-id="2b486-158">在数据流任务中，会从源提取数据、转换数据，或将数据载入数据存储。</span><span class="sxs-lookup"><span data-stu-id="2b486-158">In a data flow task, data is extracted from a source, transformed, or loaded into a data store.</span></span> <span data-ttu-id="2b486-159">一个数据流任务的输出可以是下一个数据流任务的输入，数据流可以并行运行。</span><span class="sxs-lookup"><span data-stu-id="2b486-159">The output of one data flow task can be the input to the next data flow task, and data flowss can run in parallel.</span></span> <span data-ttu-id="2b486-160">与控制流不同，无法在数据流中的任务之间添加约束。</span><span class="sxs-lookup"><span data-stu-id="2b486-160">Unlike control flows, you cannot add constraints between tasks in a data flow.</span></span> <span data-ttu-id="2b486-161">但是，可以添加数据查看器来观察每个任务处理的数据。</span><span class="sxs-lookup"><span data-stu-id="2b486-161">You can, however, add a data viewer to observe the data as it is processed by each task.</span></span>
