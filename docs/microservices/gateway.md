## <a name="what-is-an-api-gateway"></a><span data-ttu-id="aac17-109">什么是 API 网关?</span><span class="sxs-lookup"><span data-stu-id="aac17-109">What is an API gateway?</span></span>

<span data-ttu-id="aac17-110">API 网关位于客户端与服务之间。</span><span class="sxs-lookup"><span data-stu-id="aac17-110">An API gateway sits between clients and services.</span></span> <span data-ttu-id="aac17-111">它充当反向代理，将来自客户端的请求路由到服务。</span><span class="sxs-lookup"><span data-stu-id="aac17-111">It acts as a reverse proxy, routing requests from clients to services.</span></span> <span data-ttu-id="aac17-112">它还可以执行各种横切任务，例如身份验证、SSL 终止和速率限制。</span><span class="sxs-lookup"><span data-stu-id="aac17-112">It may also perform various cross-cutting tasks such as authentication, SSL termination, and rate limiting.</span></span> <span data-ttu-id="aac17-113">如果未部署网关，则客户端必须直接向前端服务发送请求。</span><span class="sxs-lookup"><span data-stu-id="aac17-113">If you don't deploy a gateway, clients must send requests directly to front-end services.</span></span> <span data-ttu-id="aac17-114">但是，直接向客户端公开服务会造成一些潜在问题：</span><span class="sxs-lookup"><span data-stu-id="aac17-114">However, there are some potential problems with exposing services directly to clients:</span></span>

- <span data-ttu-id="aac17-125">包含公共终结点的服务是潜在的受攻击面，必须得到强化。</span><span class="sxs-lookup"><span data-stu-id="aac17-125">Services with public endpoints are a potential attack surface, and must be hardened.</span></span>
