# <a name="queue-based-load-leveling-pattern"></a><span data-ttu-id="13850-104">基于队列的负载调节模式</span><span class="sxs-lookup"><span data-stu-id="13850-104">Queue-Based Load Leveling pattern</span></span>

[!INCLUDE [header](../_includes/header.md)]

<span data-ttu-id="13850-105">使用队列在任务与所调用的服务之间充当缓冲，从而缓解间歇性负载过大现象，否则会导致服务故障或任务超时。这样可以将需求高峰对任务和服务的可用性和响应能力的影响降至最低。</span><span class="sxs-lookup"><span data-stu-id="13850-105">Use a queue that acts as a buffer between a task and a service it invokes in order to smooth intermittent heavy loads that can cause the service to fail or the task to time out. This can help to minimize the impact of peaks in demand on availability and responsiveness for both the task and the service.</span></span>

## <a name="context-and-problem"></a><span data-ttu-id="13850-106">上下文和问题</span><span class="sxs-lookup"><span data-stu-id="13850-106">Context and problem</span></span>

<span data-ttu-id="13850-107">云中的许多解决方案涉及运行那些调用服务的任务。</span><span class="sxs-lookup"><span data-stu-id="13850-107">Many solutions in the cloud involve running tasks that invoke services.</span></span> <span data-ttu-id="13850-108">在这样的环境中，如果某个服务出现间歇性负载过大现象，则可能导致性能或可靠性问题。</span><span class="sxs-lookup"><span data-stu-id="13850-108">In this environment, if a service is subjected to intermittent heavy loads, it can cause performance or reliability issues.</span></span>

<span data-ttu-id="13850-109">服务可能与使用服务的任务同属一个解决方案；服务也可能是提供常用资源访问权限的第三方服务，例如缓存或存储服务。</span><span class="sxs-lookup"><span data-stu-id="13850-109">A service could be part of the same solution as the tasks that use it, or it could be a third-party service providing access to frequently used resources such as a cache or a storage service.</span></span> <span data-ttu-id="13850-110">如果多个同时运行的任务使用同一服务，则可能难以预测任意时刻向服务发出的请求的数量。</span><span class="sxs-lookup"><span data-stu-id="13850-110">If the same service is used by a number of tasks running concurrently, it can be difficult to predict the volume of requests to the service at any time.</span></span>

<span data-ttu-id="13850-111">服务可能因需求高峰而过载，无法及时响应请求。</span><span class="sxs-lookup"><span data-stu-id="13850-111">A service might experience peaks in demand that cause it to overload and be unable to respond to requests in a timely manner.</span></span> <span data-ttu-id="13850-112">向服务提交大量并发请求时，如果服务无法处理这些请求导致的争用，则也可能导致服务故障。</span><span class="sxs-lookup"><span data-stu-id="13850-112">Flooding a service with a large number of concurrent requests can also result in the service failing if it's unable to handle the contention these requests cause.</span></span>

## <a name="solution"></a><span data-ttu-id="13850-113">解决方案</span><span class="sxs-lookup"><span data-stu-id="13850-113">Solution</span></span>

<span data-ttu-id="13850-114">重构解决方案，在任务和服务之间引入队列。</span><span class="sxs-lookup"><span data-stu-id="13850-114">Refactor the solution and introduce a queue between the task and the service.</span></span> <span data-ttu-id="13850-115">任务和服务异步运行。</span><span class="sxs-lookup"><span data-stu-id="13850-115">The task and the service run asynchronously.</span></span> <span data-ttu-id="13850-116">任务将包含服务所需数据的消息发布到队列。</span><span class="sxs-lookup"><span data-stu-id="13850-116">The task posts a message containing the data required by the service to a queue.</span></span> <span data-ttu-id="13850-117">队列充当缓冲区来存储消息，直至消息被服务检索出去。</span><span class="sxs-lookup"><span data-stu-id="13850-117">The queue acts as a buffer, storing the message until it's retrieved by the service.</span></span> <span data-ttu-id="13850-118">服务从队列检索消息并对其进行处理。</span><span class="sxs-lookup"><span data-stu-id="13850-118">The service retrieves the messages from the queue and processes them.</span></span> <span data-ttu-id="13850-119">多个任务提交的请求（其生成速度可能差异很大）可以通过同一消息队列传递给服务。</span><span class="sxs-lookup"><span data-stu-id="13850-119">Requests from a number of tasks, which can be generated at a highly variable rate, can be passed to the service through the same message queue.</span></span> <span data-ttu-id="13850-120">下图显示如何使用队列调节服务的负载。</span><span class="sxs-lookup"><span data-stu-id="13850-120">This figure shows using a queue to level the load on a service.</span></span>

![图 1 - 使用队列调节服务的负载](./_images/queue-based-load-leveling-pattern.png)

<span data-ttu-id="13850-122">队列使得任务与服务分离，服务可以按自己的速度处理消息，不管并发任务提供的请求数量如何。</span><span class="sxs-lookup"><span data-stu-id="13850-122">The queue decouples the tasks from the service, and the service can handle the messages at its own pace regardless of the volume of requests from concurrent tasks.</span></span> <span data-ttu-id="13850-123">另外，即使在任务向队列发布消息时服务不可用，也不会对任务造成延迟。</span><span class="sxs-lookup"><span data-stu-id="13850-123">Additionally, there's no delay to a task if the service isn't available at the time it posts a message to the queue.</span></span>

<span data-ttu-id="13850-124">此模式具有以下优点：</span><span class="sxs-lookup"><span data-stu-id="13850-124">This pattern provides the following benefits:</span></span>

- <span data-ttu-id="13850-125">可以最大程度地提高可用性，因为服务出现延迟时，不会立即和直接影响应用程序，后者仍可继续向队列发布消息，即使服务不可用或者当前不处理消息。</span><span class="sxs-lookup"><span data-stu-id="13850-125">It can help to maximize availability because delays arising in services won't have an immediate and direct impact on the application, which can continue to post messages to the queue even when the service isn't available or isn't currently processing messages.</span></span>
- <span data-ttu-id="13850-126">可以最大程度地提高可伸缩性，因为队列数和服务数均可按需改变。</span><span class="sxs-lookup"><span data-stu-id="13850-126">It can help to maximize scalability because both the number of queues and the number of services can be varied to meet demand.</span></span>
- <span data-ttu-id="13850-127">可以控制成本，因为部署的服务实例数只需满足平均负载而非高峰负载。</span><span class="sxs-lookup"><span data-stu-id="13850-127">It can help to control costs because the number of service instances deployed only have to be adequate to meet average load rather than the peak load.</span></span>

    >  <span data-ttu-id="13850-128">当需求达到某个阈值，超出该阈值就可能导致系统故障时，某些服务会实施限制。</span><span class="sxs-lookup"><span data-stu-id="13850-128">Some services implement throttling when demand reaches a threshold beyond which the system could fail.</span></span> <span data-ttu-id="13850-129">限制可能导致可用功能减少。</span><span class="sxs-lookup"><span data-stu-id="13850-129">Throttling can reduce the functionality available.</span></span> <span data-ttu-id="13850-130">可以对这些服务实施负载调节，确保不会达到该阈值。</span><span class="sxs-lookup"><span data-stu-id="13850-130">You can implement load leveling with these services to ensure that this threshold isn't reached.</span></span>

## <a name="issues-and-considerations"></a><span data-ttu-id="13850-131">问题和注意事项</span><span class="sxs-lookup"><span data-stu-id="13850-131">Issues and considerations</span></span>

<span data-ttu-id="13850-132">在决定如何实现此模式时，请考虑以下几点：</span><span class="sxs-lookup"><span data-stu-id="13850-132">Consider the following points when deciding how to implement this pattern:</span></span>

- <span data-ttu-id="13850-133">必须实施应用程序逻辑，对服务处理消息的速度进行控制，避免目标资源过载。</span><span class="sxs-lookup"><span data-stu-id="13850-133">It's necessary to implement application logic that controls the rate at which services handle messages to avoid overwhelming the target resource.</span></span> <span data-ttu-id="13850-134">避免将需求高峰带到系统的下一阶段。</span><span class="sxs-lookup"><span data-stu-id="13850-134">Avoid passing spikes in demand to the next stage of the system.</span></span> <span data-ttu-id="13850-135">对系统进行负载测试，确保实现所需的均衡，并相应地调整队列数和处理消息的服务实例数。</span><span class="sxs-lookup"><span data-stu-id="13850-135">Test the system under load to ensure that it provides the required leveling, and adjust the number of queues and the number of service instances that handle messages to achieve this.</span></span>
- <span data-ttu-id="13850-136">消息队列是一种单向通信机制。</span><span class="sxs-lookup"><span data-stu-id="13850-136">Message queues are a one-way communication mechanism.</span></span> <span data-ttu-id="13850-137">如果任务期望从服务获得答复，则可能需要实施一种供服务用来发送响应的机制。</span><span class="sxs-lookup"><span data-stu-id="13850-137">If a task expects a reply from a service, it might be necessary to implement a mechanism that the service can use to send a response.</span></span> <span data-ttu-id="13850-138">有关详细信息，请参阅 [Asynchronous Messaging Primer](https://msdn.microsoft.com/library/dn589781.aspx)（异步消息传送入门）。</span><span class="sxs-lookup"><span data-stu-id="13850-138">For more information, see the [Asynchronous Messaging Primer](https://msdn.microsoft.com/library/dn589781.aspx).</span></span>
- <span data-ttu-id="13850-139">如果向服务应用自动缩放，而服务正在侦听队列中的请求，请谨慎操作。</span><span class="sxs-lookup"><span data-stu-id="13850-139">Be careful if you apply autoscaling to services that are listening for requests on the queue.</span></span> <span data-ttu-id="13850-140">如果这些服务共享特定的资源，上述操作可能导致资源争用加剧，降低使用队列来均衡负载的有效性。</span><span class="sxs-lookup"><span data-stu-id="13850-140">This can result in increased contention for any resources that these services share and diminish the effectiveness of using the queue to level the load.</span></span>

## <a name="when-to-use-this-pattern"></a><span data-ttu-id="13850-141">何时使用此模式</span><span class="sxs-lookup"><span data-stu-id="13850-141">When to use this pattern</span></span>

<span data-ttu-id="13850-142">此模式适用于其使用的服务可能发生过载的任何应用程序。</span><span class="sxs-lookup"><span data-stu-id="13850-142">This pattern is useful to any application that uses services that are subject to overloading.</span></span>

<span data-ttu-id="13850-143">如果应用程序需要尽量降低从服务收到响应时的延迟，则此模式不适用。</span><span class="sxs-lookup"><span data-stu-id="13850-143">This pattern isn't useful if the application expects a response from the service with minimal latency.</span></span>

## <a name="example"></a><span data-ttu-id="13850-144">示例</span><span class="sxs-lookup"><span data-stu-id="13850-144">Example</span></span>

<span data-ttu-id="13850-145">一个 Microsoft Azure Web 角色使用单独的存储服务来存储数据。</span><span class="sxs-lookup"><span data-stu-id="13850-145">A Microsoft Azure web role stores data using a separate storage service.</span></span> <span data-ttu-id="13850-146">如果该 Web 角色的大量实例同时运行，则存储服务在响应请求时，其速度可能过慢，导致这些请求超时或发生故障。</span><span class="sxs-lookup"><span data-stu-id="13850-146">If a large number of instances of the web role run concurrently, it's possible that the storage service will be unable to respond to requests quickly enough to prevent these requests from timing out or failing.</span></span> <span data-ttu-id="13850-147">下图突出显示了一个服务因来自 Web 角色实例的并发请求过多而出现过载的情况。</span><span class="sxs-lookup"><span data-stu-id="13850-147">This figure highlights a service being overwhelmed by a large number of concurrent requests from instances of a web role.</span></span>

<span data-ttu-id="13850-149">若要解决此问题，可以使用一个队列，在 Web 角色实例和存储服务之间实现负载均衡。</span><span class="sxs-lookup"><span data-stu-id="13850-149">To resolve this, you can use a queue to level the load between the web role instances and the storage service.</span></span> <span data-ttu-id="13850-150">但根据设计，存储服务是接受同步请求的，不能为了读取消息和管理吞吐量方便而轻易进行修改。</span><span class="sxs-lookup"><span data-stu-id="13850-150">However, the storage service is designed to accept synchronous requests and can't be easily modified to read messages and manage throughput.</span></span> <span data-ttu-id="13850-151">可以引入一个充当代理服务的辅助角色，让其在接收来自队列的请求后再转发给存储服务。</span><span class="sxs-lookup"><span data-stu-id="13850-151">You can introduce a worker role to act as a proxy service that receives requests from the queue and forwards them to the storage service.</span></span> <span data-ttu-id="13850-152">辅助角色中的应用程序逻辑可以控制该角色将请求传递给存储服务的速度，防止存储服务过载。</span><span class="sxs-lookup"><span data-stu-id="13850-152">The application logic in the worker role can control the rate at which it passes requests to the storage service to prevent the storage service from being overwhelmed.</span></span> <span data-ttu-id="13850-153">下图演示了如何通过队列和辅助角色在 Web 角色实例和服务之间实现负载均衡。</span><span class="sxs-lookup"><span data-stu-id="13850-153">This figure illustrates using a queue and a worker role to level the load between instances of the web role and the service.</span></span>
