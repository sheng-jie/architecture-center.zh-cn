# <a name="materialized-view-pattern"></a><span data-ttu-id="ff5c8-104">具体化视图模式</span><span class="sxs-lookup"><span data-stu-id="ff5c8-104">Materialized View pattern</span></span>

[!INCLUDE [header](../_includes/header.md)]

<span data-ttu-id="ff5c8-105">当未针对所需的查询操作完美设置数据的格式时，在一个或多个数据存储中基于数据生成预填充的视图。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-105">Generate prepopulated views over the data in one or more data stores when the data isn't ideally formatted for required query operations.</span></span> <span data-ttu-id="ff5c8-106">这有助于支持高效查询和数据提取，并可提高应用程序性能。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-106">This can help support efficient querying and data extraction, and improve application performance.</span></span>

## <a name="context-and-problem"></a><span data-ttu-id="ff5c8-107">上下文和问题</span><span class="sxs-lookup"><span data-stu-id="ff5c8-107">Context and problem</span></span>

<span data-ttu-id="ff5c8-108">存储数据时，开发人员和数据管理员的首要任务通常是专注于数据的存储方式，而不是其读取方式。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-108">When storing data, the priority for developers and data administrators is often focused on how the data is stored, as opposed to how it's read.</span></span> <span data-ttu-id="ff5c8-109">所选的存储格式通常与以下内容紧密相关：数据格式、管理数据大小和数据完整性的需求以及正在使用的存储类型。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-109">The chosen storage format is usually closely related to the format of the data, requirements for managing data size and data integrity, and the kind of store in use.</span></span> <span data-ttu-id="ff5c8-110">例如，使用 NoSQL 文档存储时，数据通常表示为一系列聚合，每个聚合包含该实体的所有信息。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-110">For example, when using NoSQL document store, the data is often represented as a series of aggregates, each containing all of the information for that entity.</span></span>

<span data-ttu-id="ff5c8-111">但是，这可能对查询产生负面影响。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-111">However, this can have a negative effect on queries.</span></span> <span data-ttu-id="ff5c8-112">如果查询仅需要某些实体的数据子集（如多个客户的订单摘要，而不需要订单的所有详细信息），它必须从相关实体中提取所有数据，以便获取所需信息。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-112">When a query only needs a subset of the data from some entities, such as a summary of orders for several customers without all of the order details, it must extract all of the data for the relevant entities in order to obtain the required information.</span></span>

## <a name="solution"></a><span data-ttu-id="ff5c8-113">解决方案</span><span class="sxs-lookup"><span data-stu-id="ff5c8-113">Solution</span></span>

<span data-ttu-id="ff5c8-114">若要支持高效查询，常见的解决方案是提前生成一个视图，该视图以适合所需结果集的格式将数据具体化。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-114">To support efficient querying, a common solution is to generate, in advance, a view that materializes the data in a format suited to the required results set.</span></span> <span data-ttu-id="ff5c8-115">具体化视图模式介绍如何在以下环境中生成预填充的数据视图：源数据的格式不适合进行查询、难以生成适合的查询或数据或数据存储的性质导致查询性能较差。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-115">The Materialized View pattern describes generating prepopulated views of data in environments where the source data isn't in a suitable format for querying, where generating a suitable query is difficult, or where query performance is poor due to the nature of the data or the data store.</span></span>

<span data-ttu-id="ff5c8-116">这些具体化视图（仅包含查询所需的数据）允许应用程序快速获取所需信息。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-116">These materialized views, which only contain data required by a query, allow applications to quickly obtain the information they need.</span></span> <span data-ttu-id="ff5c8-117">除了联接表格或合并数据实体外，具体化视图还可以包含计算列或数据项当前的值、对数据项合并值或执行转换的结果以及指定为查询的一部分的值。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-117">In addition to joining tables or combining data entities, materialized views can include the current values of calculated columns or data items, the results of combining values or executing transformations on the data items, and values specified as part of the query.</span></span> <span data-ttu-id="ff5c8-118">甚至可以仅针对单个查询优化具体化视图。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-118">A materialized view can even be optimized for just a single query.</span></span>

<span data-ttu-id="ff5c8-119">关键点在于，具体化视图可从源数据存储完全重新生成，所以它及其包含的信息是完全可释放的。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-119">A key point is that a materialized view and the data it contains is completely disposable because it can be entirely rebuilt from the source data stores.</span></span> <span data-ttu-id="ff5c8-120">具体化视图从不通过应用程序直接更新，因此它是专用缓存。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-120">A materialized view is never updated directly by an application, and so it's a specialized cache.</span></span>

<span data-ttu-id="ff5c8-121">视图的源数据更改时，必须更新视图以使其包含新的信息。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-121">When the source data for the view changes, the view must be updated to include the new information.</span></span> <span data-ttu-id="ff5c8-122">可计划自动执行此操作，或在系统检测到原始数据发生更改时执行操作。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-122">You can schedule this to happen automatically, or when the system detects a change to the original data.</span></span> <span data-ttu-id="ff5c8-123">在某些情况下，可能需要手动重新生成视图。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-123">In some cases it might be necessary to regenerate the view manually.</span></span> <span data-ttu-id="ff5c8-124">下图中的示例演示了可以如何使用具体化视图模式。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-124">The figure shows an example of how the Materialized View pattern might be used.</span></span>

## <a name="issues-and-considerations"></a><span data-ttu-id="ff5c8-126">问题和注意事项</span><span class="sxs-lookup"><span data-stu-id="ff5c8-126">Issues and considerations</span></span>

<span data-ttu-id="ff5c8-127">在决定如何实现此模式时，请考虑以下几点：</span><span class="sxs-lookup"><span data-stu-id="ff5c8-127">Consider the following points when deciding how to implement this pattern:</span></span>

<span data-ttu-id="ff5c8-128">视图更新的方式和时间。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-128">How and when the view will be updated.</span></span> <span data-ttu-id="ff5c8-129">理想情况下，当有事件指示源数据发生更改时，会重新生成视图（尽管源数据更改速度过快时会导致过度开销）。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-129">Ideally it'll regenerate in response to an event indicating a change to the source data, although this can lead to excessive overhead if the source data changes rapidly.</span></span> <span data-ttu-id="ff5c8-130">或者，可以考虑使用计划的任务、外部触发器或通过手动操作重新生成视图。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-130">Alternatively, consider using a scheduled task, an external trigger, or a manual action to regenerate the view.</span></span>

<span data-ttu-id="ff5c8-131">在某些系统（例如，使用事件溯源模式维护仅含修改数据的事件的存储时）中，必须使用具体化视图。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-131">In some systems, such as when using the Event Sourcing pattern to maintain a store of only the events that modified the data, materialized views are necessary.</span></span> <span data-ttu-id="ff5c8-132">通过检查所有事件确定当前状态来预填充视图，这可能是从事件存储获取信息的唯一方法。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-132">Prepopulating views by examining all events to determine the current state might be the only way to obtain information from the event store.</span></span> <span data-ttu-id="ff5c8-133">如果没有使用事件溯源，则需考虑具体化视图对你而言是否有用。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-133">If you're not using Event Sourcing, you need to consider whether a materialized view is helpful or not.</span></span> <span data-ttu-id="ff5c8-134">具体化视图通常专门用于一个或少量查询的情况。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-134">Materialized views tend to be specifically tailored to one, or a small number of queries.</span></span> <span data-ttu-id="ff5c8-135">如果使用了大量查询，具体化视图会产生不可接受的存储容量需求和存储成本。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-135">If many queries are used, materialized views can result in unacceptable storage capacity requirements and storage cost.</span></span>

<span data-ttu-id="ff5c8-136">如果此情况定期出现，请在生成视图以及更新视图时考虑这对数据一致性的影响。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-136">Consider the impact on data consistency when generating the view, and when updating the view if this occurs on a schedule.</span></span> <span data-ttu-id="ff5c8-137">如果在生成视图时源数据发生更改，则此视图中的数据副本不会与原始数据完全一致。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-137">If the source data is changing at the point when the view is generated, the copy of the data in the view won't be fully consistent with the original data.</span></span>

<span data-ttu-id="ff5c8-138">请考虑视图的存储位置。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-138">Consider where you'll store the view.</span></span> <span data-ttu-id="ff5c8-139">视图所在的存储或分区无需与原始数据的相同。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-139">The view doesn't have to be located in the same store or partition as the original data.</span></span> <span data-ttu-id="ff5c8-140">它可以是几个不同分区合并形成的子集。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-140">It can be a subset from a few different partitions combined.</span></span>

<span data-ttu-id="ff5c8-141">如果视图丢失，可重新生成视图。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-141">A view can be rebuilt if lost.</span></span> <span data-ttu-id="ff5c8-142">因此，如果视图是暂时性视图，且仅用于通过反映数据的当前状态提升查询性能或提高可伸缩性，则可将其存储在缓存或可靠性较低的位置。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-142">Because of that, if the view is transient and is only used to improve query performance by reflecting the current state of the data, or to improve scalability, it can be stored in a cache or in a less reliable location.</span></span>

<span data-ttu-id="ff5c8-143">定义具体化视图时，基于以下内容将数据项或列添加到具体化视图使值最大化：现有数据项的计算或转换、在查询中传递的值或这些值在适当情况下的组合。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-143">When defining a materialized view, maximize its value by adding data items or columns to it based on computation or transformation of existing data items, on values passed in the query, or on combinations of these values when appropriate.</span></span>

<span data-ttu-id="ff5c8-144">如果存储机制支持，请考虑将具体化视图编制索引，进一步提高性能。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-144">Where the storage mechanism supports it, consider indexing the materialized view to further increase performance.</span></span> <span data-ttu-id="ff5c8-145">大多数关系数据库和基于 Apache Hadoop 的大数据解决方案都支持为视图编制索引。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-145">Most relational databases support indexing for views, as do big data solutions based on Apache Hadoop.</span></span>

## <a name="when-to-use-this-pattern"></a><span data-ttu-id="ff5c8-146">何时使用此模式</span><span class="sxs-lookup"><span data-stu-id="ff5c8-146">When to use this pattern</span></span>

<span data-ttu-id="ff5c8-147">此模式适合用于：</span><span class="sxs-lookup"><span data-stu-id="ff5c8-147">This pattern is useful when:</span></span>
- <span data-ttu-id="ff5c8-148">对难以直接查询的数据创建具体化视图，或查询必须十分复杂才能提取以规范化、半结构化或非结构化方式存储的数据。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-148">Creating materialized views over data that's difficult to query directly, or where queries must be very complex to extract data that's stored in a normalized, semi-structured, or unstructured way.</span></span>
- <span data-ttu-id="ff5c8-149">创建的临时视图有以下作用时：可极大地提高查询性能，或可直接充当 UI、报告或显示的源视图或数据传输对象。</span><span class="sxs-lookup"><span data-stu-id="ff5c8-149">Creating temporary views that can dramatically improve query performance, or can act directly as source views or data transfer objects for the UI, for reporting, or for display.</span></span>
