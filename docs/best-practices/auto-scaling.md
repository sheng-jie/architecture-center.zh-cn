# <a name="autoscaling"></a><span data-ttu-id="35d98-103">自动缩放</span><span class="sxs-lookup"><span data-stu-id="35d98-103">Autoscaling</span></span>
[!INCLUDE [header](../_includes/header.md)]

<span data-ttu-id="35d98-104">自动缩放是动态分配资源以满足性能需求的过程。</span><span class="sxs-lookup"><span data-stu-id="35d98-104">Autoscaling is the process of dynamically allocating resources to match performance requirements.</span></span> <span data-ttu-id="35d98-105">当工作量增大时，应用程序可能需要额外的资源来维持所需的性能级别和满足服务级别协议 (SLA)。</span><span class="sxs-lookup"><span data-stu-id="35d98-105">As the volume of work grows, an application may need additional resources to maintain the desired performance levels and satisfy service-level agreements (SLAs).</span></span> <span data-ttu-id="35d98-106">当需求降低，不再需要额外的资源时，可以取消分配资源，最大程度地降低成本。</span><span class="sxs-lookup"><span data-stu-id="35d98-106">As demand slackens and the additional resources are no longer needed, they can be de-allocated to minimize costs.</span></span>

<span data-ttu-id="35d98-107">自动缩放可以利用云托管环境的弹性，同时可以降低管理开销。</span><span class="sxs-lookup"><span data-stu-id="35d98-107">Autoscaling takes advantage of the elasticity of cloud-hosted environments while easing management overhead.</span></span> <span data-ttu-id="35d98-108">它可以减少操作员持续监视系统性能，并做出有关添加或删除资源的决策的需要。</span><span class="sxs-lookup"><span data-stu-id="35d98-108">It reduces the need for an operator to continually monitor the performance of a system and make decisions about adding or removing resources.</span></span>

<span data-ttu-id="35d98-109">应用程序缩放有两种主要的方式：</span><span class="sxs-lookup"><span data-stu-id="35d98-109">There are two main ways that an application can scale:</span></span> 

* <span data-ttu-id="35d98-110">垂直缩放，也称为增加和减少，表示改变资源的容量。</span><span class="sxs-lookup"><span data-stu-id="35d98-110">**Vertical scaling**, also called scaling up and down, means changing the capacity of a resource.</span></span> <span data-ttu-id="35d98-111">例如，可将应用程序移动到更大的 VM 中。</span><span class="sxs-lookup"><span data-stu-id="35d98-111">For example, you could move an application to a larger VM size.</span></span> <span data-ttu-id="35d98-112">垂直缩放在重新部署时通常会要求系统暂时不可用。</span><span class="sxs-lookup"><span data-stu-id="35d98-112">Vertical scaling often requires making the system temporarily unavailable while it is being redeployed.</span></span> <span data-ttu-id="35d98-113">因此，自动化垂直缩放并不常见。</span><span class="sxs-lookup"><span data-stu-id="35d98-113">Therefore, it's less common to automate vertical scaling.</span></span>
* <span data-ttu-id="35d98-114">水平缩放，也称为扩大和缩小，表示添加或删除资源的实例。</span><span class="sxs-lookup"><span data-stu-id="35d98-114">**Horizontal scaling**, also called scaling out and in, means adding or removing instances of a resource.</span></span> <span data-ttu-id="35d98-115">在预配新资源时，应用程序无需中断，可持续运行。</span><span class="sxs-lookup"><span data-stu-id="35d98-115">The application continues running without interruption as new resources are provisioned.</span></span> <span data-ttu-id="35d98-116">当预配过程完成时，解决方案就已部署在这些额外资源上。</span><span class="sxs-lookup"><span data-stu-id="35d98-116">When the provisioning process is complete, the solution is deployed on these additional resources.</span></span> <span data-ttu-id="35d98-117">如果需求降低，额外的资源可以完全关闭并解除分配。</span><span class="sxs-lookup"><span data-stu-id="35d98-117">If demand drops, the additional resources can be shut down cleanly and deallocated.</span></span> 

<span data-ttu-id="35d98-118">许多基于云的系统（包括 Microsoft Azure）支持自动水平缩放。</span><span class="sxs-lookup"><span data-stu-id="35d98-118">Many cloud-based systems, including Microsoft Azure, support automatic horizontal scaling.</span></span> <span data-ttu-id="35d98-119">本文的余下内容重点介绍水平缩放。</span><span class="sxs-lookup"><span data-stu-id="35d98-119">The rest of this article focuses on horizontal scaling.</span></span>

> [!NOTE]
> <span data-ttu-id="35d98-120">自动缩放主要适用于计算资源。</span><span class="sxs-lookup"><span data-stu-id="35d98-120">Autoscaling mostly applies to compute resources.</span></span> <span data-ttu-id="35d98-121">虽然可以水平缩放数据库或消息队列，但这通常涉及非自动的[数据分区][data-partitioning]。</span><span class="sxs-lookup"><span data-stu-id="35d98-121">While it's possible to horizontally scale a database or message queue, this usually involves [data partitioning][data-partitioning], which is generally not automated.</span></span>
