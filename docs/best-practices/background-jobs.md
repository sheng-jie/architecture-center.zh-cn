# <a name="background-jobs"></a><span data-ttu-id="48b82-103">后台作业</span><span class="sxs-lookup"><span data-stu-id="48b82-103">Background jobs</span></span>
[!INCLUDE [header](../_includes/header.md)]

<span data-ttu-id="48b82-104">许多类型的应用程序需要运行与用户界面 (UI) 无关的后台任务。</span><span class="sxs-lookup"><span data-stu-id="48b82-104">Many types of applications require background tasks that run independently of the user interface (UI).</span></span> <span data-ttu-id="48b82-105">示例包括批处理作业、密集处理型任务，以及长时间运行的过程，例如工作流。</span><span class="sxs-lookup"><span data-stu-id="48b82-105">Examples include batch jobs, intensive processing tasks, and long-running processes such as workflows.</span></span> <span data-ttu-id="48b82-106">后台作业无需用户交互就可执行 -- 应用程序可以启动作业，并继续处理来自用户的交互请求。</span><span class="sxs-lookup"><span data-stu-id="48b82-106">Background jobs can be executed without requiring user interaction--the application can start the job and then continue to process interactive requests from users.</span></span> <span data-ttu-id="48b82-107">这有助于减少应用程序 UI 上的负载，从而提高可用性，降低交互响应时间。</span><span class="sxs-lookup"><span data-stu-id="48b82-107">This can help to minimize the load on the application UI, which can improve availability and reduce interactive response times.</span></span>

<span data-ttu-id="48b82-108">例如，如果应用程序需要生成用户上传的图像缩图，可以后台作业的形式执行此操作，并在完成时会缩图保存到存储中，用户无需等待过程完成。</span><span class="sxs-lookup"><span data-stu-id="48b82-108">For example, if an application is required to generate thumbnails of images that are uploaded by users, it can do this as a background job and save the thumbnail to storage when it is complete--without the user needing to wait for the process to be completed.</span></span> <span data-ttu-id="48b82-109">同样地，下单的用户可以启动一个后台工作流来处理订单，同时，UI 可让用户继续浏览 Web 应用。</span><span class="sxs-lookup"><span data-stu-id="48b82-109">In the same way, a user placing an order can initiate a background workflow that processes the order, while the UI allows the user to continue browsing the web app.</span></span> <span data-ttu-id="48b82-110">当后台作业完成时，可以更新存储的订单数据，并将确认订单的电子邮件发送给用户。</span><span class="sxs-lookup"><span data-stu-id="48b82-110">When the background job is complete, it can update the stored orders data and send an email to the user that confirms the order.</span></span>

<span data-ttu-id="48b82-111">考虑是否将任务作为后台作业执行时，主要的准则是：是否可以在无需用户交互的情况下运行该任务，且 UI 无需等待作业完成。</span><span class="sxs-lookup"><span data-stu-id="48b82-111">When you consider whether to implement a task as a background job, the main criteria is whether the task can run without user interaction and without the UI needing to wait for the job to be completed.</span></span> <span data-ttu-id="48b82-112">在完成之前需要用户或 UI 等待的任务不适合作为后台作业运行。</span><span class="sxs-lookup"><span data-stu-id="48b82-112">Tasks that require the user or the UI to wait while they are completed might not be appropriate as background jobs.</span></span>

## <a name="types-of-background-jobs"></a><span data-ttu-id="48b82-113">后台作业的类型</span><span class="sxs-lookup"><span data-stu-id="48b82-113">Types of background jobs</span></span>

* <span data-ttu-id="48b82-115">CPU 密集型作业，例如数学计算或结构模型分析。</span><span class="sxs-lookup"><span data-stu-id="48b82-115">CPU-intensive jobs, such as mathematical calculations or structural model analysis.</span></span>
* <span data-ttu-id="48b82-116">I/O 密集型作业，例如执行一系列存储事务或文件索引编制。</span><span class="sxs-lookup"><span data-stu-id="48b82-116">I/O-intensive jobs, such as executing a series of storage transactions or indexing files.</span></span>
* <span data-ttu-id="48b82-117">批处理作业，例如夜间数据更新或计划的处理。</span><span class="sxs-lookup"><span data-stu-id="48b82-117">Batch jobs, such as nightly data updates or scheduled processing.</span></span>
* <span data-ttu-id="48b82-118">长时间运行的工作流，例如订单履行或预配服务和系统。</span><span class="sxs-lookup"><span data-stu-id="48b82-118">Long-running workflows, such as order fulfillment, or provisioning services and systems.</span></span>
* <span data-ttu-id="48b82-119">敏感数据处理，其中的任务将转移到更安全的位置以进行处理。</span><span class="sxs-lookup"><span data-stu-id="48b82-119">Sensitive-data processing where the task is handed off to a more secure location for processing.</span></span> <span data-ttu-id="48b82-120">例如，可能不希望处理 Web 应用中的敏感数据。</span><span class="sxs-lookup"><span data-stu-id="48b82-120">For example, you might not want to process sensitive data within a web app.</span></span> <span data-ttu-id="48b82-121">而想使用[守护程序](http://msdn.microsoft.com/library/dn589793.aspx)等模式将数据传输到有权访问受保护存储的已隔离后台进程。</span><span class="sxs-lookup"><span data-stu-id="48b82-121">Instead, you might use a pattern such as [Gatekeeper](http://msdn.microsoft.com/library/dn589793.aspx) to transfer the data to an isolated background process that has access to protected storage.</span></span>

## <a name="triggers"></a><span data-ttu-id="48b82-122">触发器</span><span class="sxs-lookup"><span data-stu-id="48b82-122">Triggers</span></span>
<span data-ttu-id="48b82-123">可通过多种不同的方式启动后台作业。</span><span class="sxs-lookup"><span data-stu-id="48b82-123">Background jobs can be initiated in several different ways.</span></span> <span data-ttu-id="48b82-124">这些方式属于以下类别之一：</span><span class="sxs-lookup"><span data-stu-id="48b82-124">They fall into one of the following categories:</span></span>

### <a name="event-driven-triggers"></a><span data-ttu-id="48b82-130">事件驱动的触发器</span><span class="sxs-lookup"><span data-stu-id="48b82-130">Event-driven triggers</span></span>

* <span data-ttu-id="48b82-133">UI 或另一个作业将消息放入队列。</span><span class="sxs-lookup"><span data-stu-id="48b82-133">The UI or another job places a message in a queue.</span></span> <span data-ttu-id="48b82-134">该消息包含有关已执行的操作的数据，例如下单的用户。</span><span class="sxs-lookup"><span data-stu-id="48b82-134">The message contains data about an action that has taken place, such as the user placing an order.</span></span> <span data-ttu-id="48b82-135">后台任务将侦听此队列，并检测新消息是否已到达。</span><span class="sxs-lookup"><span data-stu-id="48b82-135">The background task listens on this queue and detects the arrival of a new message.</span></span> <span data-ttu-id="48b82-136">它将读取消息，并将其中的数据用作后台作业的输入。</span><span class="sxs-lookup"><span data-stu-id="48b82-136">It reads the message and uses the data in it as the input to the background job.</span></span>

* <span data-ttu-id="48b82-140">UI 或另一个作业向终结点（例如 HTTPS URI，或作为 Web 服务公开的 API）发出请求。</span><span class="sxs-lookup"><span data-stu-id="48b82-140">The UI or another job makes a request to an endpoint, such as an HTTPS URI, or an API that is exposed as a web service.</span></span> <span data-ttu-id="48b82-141">它在请求的过程中传递所需的数据以完成后台任务。</span><span class="sxs-lookup"><span data-stu-id="48b82-141">It passes the data that is required to complete the background task as part of the request.</span></span> <span data-ttu-id="48b82-142">终结点或 Web 服务调用后台任务，将数据用作其输入。</span><span class="sxs-lookup"><span data-stu-id="48b82-142">The endpoint or web service invokes the background task, which uses the data as its input.</span></span>

<span data-ttu-id="48b82-143">适合事件驱动调用的任务的典型示例包括图像处理、工作流、将信息发送到远程服务、发送电子邮件消息，以及在多租户应用程序中预配新用户。</span><span class="sxs-lookup"><span data-stu-id="48b82-143">Typical examples of tasks that are suited to event-driven invocation include image processing, workflows, sending information to remote services, sending email messages, and provisioning new users in multitenant applications.</span></span>

### <a name="schedule-driven-triggers"></a><span data-ttu-id="48b82-144">计划驱动的触发器</span><span class="sxs-lookup"><span data-stu-id="48b82-144">Schedule-driven triggers</span></span>

<span data-ttu-id="48b82-157">后台作业通过 UI 或调用后台任务的进程，以异步方式在独立进程甚至不同的位置运行。</span><span class="sxs-lookup"><span data-stu-id="48b82-157">Background jobs execute asynchronously in a separate process, or even in a separate location, from the UI or the process that invoked the background task.</span></span> <span data-ttu-id="48b82-158">在理想情况下，后台任务是“即发即弃”的操作，其执行进度不会影响 UI 或调用进程。</span><span class="sxs-lookup"><span data-stu-id="48b82-158">Ideally, background tasks are “fire and forget” operations, and their execution progress has no impact on the UI or the calling process.</span></span> <span data-ttu-id="48b82-159">这意味着，调用进程不会等待任务完成。</span><span class="sxs-lookup"><span data-stu-id="48b82-159">This means that the calling process does not wait for completion of the tasks.</span></span> <span data-ttu-id="48b82-160">因此无法自动检测任务的结束时间。</span><span class="sxs-lookup"><span data-stu-id="48b82-160">Therefore, it cannot automatically detect when the task ends.</span></span>

* <span data-ttu-id="48b82-172">让后台任务通过 API 回调 UI 或调用方，以指示预定义时间点或完成时的状态。</span><span class="sxs-lookup"><span data-stu-id="48b82-172">Have the background task call back to the UI or caller through an API to indicate status at predefined points or on completion.</span></span> <span data-ttu-id="48b82-173">这可以通过本地引发的事件或通过发布与订阅机制来实现。</span><span class="sxs-lookup"><span data-stu-id="48b82-173">This might be through events raised locally or through a publish-and-subscribe mechanism.</span></span> <span data-ttu-id="48b82-174">可在请求或事件负载中包含后台任务必须返回给调用方的数据。</span><span class="sxs-lookup"><span data-stu-id="48b82-174">Data that the background task must return to the caller can be included in the request or event payload.</span></span>

  * <span data-ttu-id="48b82-467">一般而言，后台任务会在队列中扫视消息，这会暂时向其他消息使用者隐藏消息。</span><span class="sxs-lookup"><span data-stu-id="48b82-467">Typically, a background task will peek at messages in the queue, which temporarily hides them from other message consumers.</span></span> <span data-ttu-id="48b82-468">然后，它会在成功处理消息后，将其删除。</span><span class="sxs-lookup"><span data-stu-id="48b82-468">Then it deletes the messages after they have been successfully processed.</span></span> <span data-ttu-id="48b82-469">如果后台任务在处理某个消息时失败，该消息会在扫视超时后重新出现在队列中。</span><span class="sxs-lookup"><span data-stu-id="48b82-469">If a background task fails when processing a message, that message will reappear on the queue after the peek time-out expires.</span></span> <span data-ttu-id="48b82-470">该消息由任务的另一个实例处理，或在此实例的下一个处理周期进行处理。</span><span class="sxs-lookup"><span data-stu-id="48b82-470">It will be processed by another instance of the task or during the next processing cycle of this instance.</span></span> <span data-ttu-id="48b82-471">如果消息一直导致使用者出错，则会阻止任务、队列，并最终在队列填满时阻止应用程序本身。</span><span class="sxs-lookup"><span data-stu-id="48b82-471">If the message consistently causes an error in the consumer, it will block the task, the queue, and eventually the application itself when the queue becomes full.</span></span> <span data-ttu-id="48b82-472">因此，请务必在队列中检测并删除有害消息。</span><span class="sxs-lookup"><span data-stu-id="48b82-472">Therefore, it is vital to detect and remove poison messages from the queue.</span></span> <span data-ttu-id="48b82-473">如果使用 Azure 服务总线，导致出错的消息可以自动或手动移到关联的死信队列。</span><span class="sxs-lookup"><span data-stu-id="48b82-473">If you are using Azure Service Bus, messages that cause an error can be moved automatically or manually to an associated dead letter queue.</span></span>
